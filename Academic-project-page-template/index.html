Abstract

This paper present FORMIGA as an innovative solution for enhancing human-mediated robot autonomy under challenging field applications. FORMIGA emphasises operational management, seamless integration, and sophisticated automation. This paper surveys the state of the art on fleet management systems, including commercially available solutions, so as to establish a baseline around the topic. Afterwards, the paper delves into FORMIGA’s features and its pivotal role within the FEROX project - an European project aimed at revolutionising wild berry picking using robots to improve working conditions and operational eﬀiciency. Through simulations, we assess FORMIGA’s impact, demonstrating its capability to adapt to varying situations and its potential to generate collaborative tasks using a large language model. Our findings underscore FORMIGA’s contribution to the future of human-centric robotic services, marking a significant step towards realising flexible human-robot ecosystems in challenging domains.

FORMIGA Overview

FORMIGA is a cutting-edge Fleet Management System (FMS) designed to coordinate the operations of both robotic and human agents in dynamic and complex environments. It provides a versatile platform for managing multi-agent systems, optimizing resource deployment, and ensuring seamless interaction between robots and human operators. The development of FORMIGA followed a structured design process, beginning with the foundational ROOSTER framework. FORMIGA builds on this foundation by incorporating additional features tailored to the specific needs of challenging field applications involving both humans and robots.

The architecture of FORMIGA integrates several key components, including:

  • ROS: FORMIGA utilizes ROS to facilitate seamless communication between agents (both human and robotic), ensuring a cohesive operational framework.
  • User Interface: FORMIGA’s graphical interface allows operators to manage fleets of agents, monitor real-time system status, and configure agents (e.g., defining key ROS Actions).
  • User-tailored Task Coding: FORMIGA enables task coding as collections of ROS Action Clients connected to the Action Servers enabled by agents.
  • Task Management: Building on ROOSTER’s task management system, FORMIGA offers a robust framework for defining, allocating, and executing tasks, enabling multi-agent collaboration and dynamic resource optimization.
  • LLM-generated Task Coding: FORMIGA leverages a Large Language Model (LLM) to simplify and reduce the effort required for task coding.

Task Coding in FORMIGA

Task coding in FORMIGA involves creating custom Python functions to define and execute specific tasks for both robots and humans. These tasks are composed of multiple ROS action clients, which send goals to agents, ensuring coordinated fleet operations. During execution, FORMIGA continuously updates the status of all agents involved, facilitating seamless multitasking in dynamic environments.

A typical task function starts by converting user inputs into actionable goals, selecting the appropriate robot, and sending commands to perform actions. FORMIGA allows for both user-specified robot selection or automatic selection based on availability and task requirements. Actions are executed sequentially or concurrently, with the system managing task failures and resource allocation in the background.

LLM-Generated Task Coding in FORMIGA

The integration of a large language model (LLM) within FORMIGA enhances the task coding process, enabling the system to generate Python-based task functions. By leveraging cloud-based LLM platforms such as the Groq platform in combination with the LLaMA 3 model, FORMIGA achieves the computational efficiency necessary for real-time task generation without requiring costly local hardware setups.

The LLM is fine-tuned to the specific context of FORMIGA's operations, allowing it to generate reliable and context-aware Python code. The model learns coding conventions and action primitives, such as moving, patrolling, or loading, enabling seamless interaction with ROS action servers for task execution. Once fine-tuned, the model can create task functions that were not pre-defined, mapping user goals to executable agent actions.
